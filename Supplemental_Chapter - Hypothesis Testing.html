Hypothesis Testing

Hypothesis testing is a way to decide if a certain statement about a population might be true based on sample data.

Example:

Let’s say we want to know if the average pollution in a set of water samples is above the legal limit. Or if young deer in a region are, on average, healthy.

Step 1: Define Your Hypotheses

First, we need to define two hypotheses: the research hypothesis and the null hypothesis.

  • Research Hypothesis (Ha): This is what we aim to support. Remember, we can never “prove” Ha, only fail to reject H0. It can take a few forms based on the question:
    • Ha: average pollution > legal limit (pollution is too high)
    • Ha: average pollution < legal limit (pollution is too low)
    • Ha: average pollution ≠ legal limit (pollution is just different)
  • Null Hypothesis (H0): This is the default or ‘no change’ scenario. It’s opposite to the research hypothesis.
    • H0: average pollution ≤ legal limit (for the first Ha)
    • H0: average pollution ≥ legal limit (for the second Ha)
    • H0: average pollution = legal limit (for the third Ha)

Step 2: Choose Your Test Statistic

Based on the data, we’ll compute a test statistic. This number will help us decide which hypothesis seems more likely.

Step 3: Determine the Rejection Region

Before running the test, we decide on a rejection region. If our test statistic falls in this region, we’ll reject the null hypothesis.

Step 4: Check Assumptions

Before drawing conclusions, ensure that the test’s conditions and assumptions are satisfied.

Step 5: Draw Conclusions

Finally, based on the test statistic and the rejection region, decide whether to reject the null hypothesis.

Errors in Hypothesis Testing

Even with all this, sometimes we make wrong decisions.

  • Type I Error (α): We reject the null hypothesis when it’s actually true. It’s like accusing an innocent person of a crime. Commonly, α is set to 0.05 (or 5%).

  • Type II Error (β): We accept the null hypothesis when it’s actually false. Like letting a guilty person go free.

Decision Table for Hypothesis Testing
Decision If.the.null.hypothesis.is.True If.the.null.hypothesis.is.False
Reject H0 Type I error (prob = α) Correct (prob = 1 - β)
Fail to reject H0 Correct (prob = 1 - α) Type II error (prob = β)

Remember: When α decreases, β increases and vice-versa.

One Tail or Two?

In our pollution example, suppose the historic pollution level was 10.0 ppb. Our hypotheses can look like:

  • H0: pollution = 10.0 ppb (Two-tailed: we’re checking if pollution is either significantly more or less than 10 ppb)
  • H0: pollution < 10.0 ppb (One-tailed: we’re only checking if pollution is significantly more than 10 ppb)
  • H0: pollution > 10.0 ppb (One-tailed: we’re only checking if pollution is significantly less than 10 ppb)

Interpreting Significance Using P-values

The p-value is a probability that tells us how unusual our data would be if the null hypothesis were true.

  • One-Tailed Test: The p-value tells us the chance of an average as extreme as our sample’s, if the null hypothesis is true.
  • Two-Tailed Test: It tells us the chance of an average as different from the null hypothesis as our sample’s.

Decision Rule: If p-value < α, then we reject the null hypothesis.

How to Test the Null Hypothesis?

  1. Confidence Interval Method: If your confidence interval includes the null value, don’t reject the hypothesis.
  2. Rejection Region Method: If the test statistic lies in the rejection region, reject the hypothesis.
  3. P-value Method: If p-value < α, reject the hypothesis.

Pro Tip: When reporting your findings, use a results statement. For example: “The average mercury level near the smelter is higher than the limit (z=2.85, n=32, p=0.004).”
